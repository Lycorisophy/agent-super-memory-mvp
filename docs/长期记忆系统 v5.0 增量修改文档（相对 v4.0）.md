# 长期记忆系统 v5.0 增量修改文档（相对 v4.0）

## 1. 概述

v5.0 在 v4.0 统一存储格式的基础上，针对**结构化过滤能力不足**的问题进行了增强。主要改进点：

- 为 Neo4j 的 `Memory` 节点增加 `tense`（时态）和 `confidence`（置信度）可选字段，支持精确筛选。
- 优化工具接口参数，允许大模型在存储时传递这些字段。
- 调整系统提示词，明确要求大模型识别并填写时态和置信度。

**v4.0 已存储的数据不受影响**，新字段为空时保持向后兼容。

---

## 2. 表结构变更

### 2.1 Neo4j 节点属性变更

**Memory 节点原有属性**（v4.0）：
| 属性名 | 类型 |
| :--- | :--- |
| `memory_id` | 字符串 |
| `content` | 字符串 |
| `timestamp` | 整数 |
| `memory_type` | 字符串 |

**v5.0 新增属性**：
| 属性名 | 中文含义 | 类型 | 取值说明 | 是否必填 |
| :--- | :--- | :--- | :--- | :--- |
| `tense` | 时态 | 字符串 | `past`（过去）、`present`（现在）、`future`（将来） | 可选 |
| `confidence` | 置信度 | 字符串 | `real`（真实）、`imagined`（虚构）、`planned`（计划） | 可选 |

**升级 Cypher 语句**（无需删除重建，直接添加属性）：
```cypher
// 新增约束（可选，仅用于确保 memory_id 唯一性）
CREATE CONSTRAINT memory_id_unique IF NOT EXISTS FOR (m:Memory) REQUIRE m.memory_id IS UNIQUE;

// 已有节点自动获得新属性（值为 null），无需执行额外语句
```

### 2.2 Milvus 集合无变更

v5.0 不修改 Milvus 集合结构，因为向量检索仍基于 `content` 文本，新增的 `tense` 和 `confidence` 仅用于 Neo4j 侧的结构化过滤。

## 3. 工具接口变更

### 3.1 `store_memory` 工具参数调整

在 `memories` 数组的每个元素中，增加两个可选字段：

**v4.0 格式**：
```json

  "time": "2026-04-21 14:30",
  "location": "沃尔玛",
  "subject": "我",
  "content": "与李丽购买排骨",
  "source": "..."
}
```

**v5.0 新增字段**：
```json
{
  "type": "event",
  "time": "2026-04-21 14:30",
  "location": "沃尔玛",
  "subject": "我",
  "content": "与李丽购买排骨",
  "source": "...",
  "tense": "past",        // 新增：past / present / future
  "confidence": "real"    // 新增：real / imagined / planned
}
```

完整 JSON Schema 更新部分：
```json
{
  "properties": {
    "memories": {
      "items": {
        "properties": {
          // ... 原有字段保持不变
          "tense": { 
            "type": "string", 
            "enum": ["past", "present", "future"],
            "description": "时态，不明确可不填" 
          },
          "confidence": { 
            "type": "string", 
            "enum": ["real", "imagined", "planned"],
            "description": "置信度，不明确可不填" 
          }
        }
      }
    }
  }
}
```

## 4. 系统提示词变更

### 4.1 存储侧提示词（新增部分）

在“提取规范”的**事件**条目下增加说明：

```text
**1. 事件**
- 必填字段：`type`、`content`。
- 其他字段尽量补全，若用户未提及则留空：
  ...
  - `tense`：时态，可选 `past`（过去）、`present`（现在）、`future`（将来）。
  - `confidence`：置信度，可选 `real`（真实发生）、`imagined`（想象/梦境/虚构）、`planned`（计划中）。
- 对于虚拟语气（如“我梦见...”）、未来计划（如“我打算...”），请正确填写时态和置信度。
```

### 4.2 查询侧提示词（新增部分）

在“查询参数构造指南”中增加过滤说明：

```text
**5. 结构化过滤**
- 若用户明确限定时间属性（如“过去的经历”、“计划中的事”、“梦里的事”），可设置 `tense` 或 `confidence` 进行精确筛选。
```

## 5. 代码改动要点

### 5.1 `store_memory` 函数修改

存储记忆时，从输入参数中读取 `tense` 和 `confidence` 并写入 Neo4j：

```python
def store_memory(self, user_id: str, memories: List[Dict], relations: List[Dict] = None):
    for mem in memories:
        memory_id = f"mem_{self._id_gen.next_str()}"
        content = self._build_content(mem)  # 按模板拼接文本
        timestamp = self._parse_time(mem.get("time"))
        memory_type = mem["type"]
        tense = mem.get("tense")      # 新增
        confidence = mem.get("confidence")  # 新增

        session.run("""
            MATCH (u:User {user_id: $user_id})
            CREATE (m:Memory {
                memory_id: $memory_id,
                content: $content,
                timestamp: $timestamp,
                memory_type: $memory_type,
                tense: $tense,
                confidence: $confidence
            })
            CREATE (u)-[:HAS_MEMORY]->(m)
        """, ...)
```

### 5.2 `query_memory` 函数修改

在 Neo4j 查询中增加可选过滤条件：

```python
def query_memory(self, user_id: str, query_text: str, memory_types: List[str] = None,
                 tense: str = None, confidence: str = None, ...):
    # 向量检索 Milvus 获取候选 memory_id
    # 构造 Cypher 查询时添加过滤
    where_clauses = ["m.memory_id IN $candidate_ids"]
    if memory_types:
        where_clauses.append("m.memory_type IN $memory_types")
    if tense:
        where_clauses.append("m.tense = $tense")
    if confidence:
        where_clauses.append("m.confidence = $confidence")
    
    query = f"MATCH (m:Memory) WHERE {' AND '.join(where_clauses)} RETURN m.content"
    ...
```

## 7. 升级步骤总结

1. **更新代码**：按第5节修改 `store_memory` 和 `query_memory` 函数。
2. **更新工具 Schema**：调整 Function Calling 定义。
3. **更新提示词**：按第4节补充时态、置信度提取说明。
4. **Neo4j 数据库**：执行第2.1节的约束语句（无需重建表，新字段自动生效）。
5. **重启服务**：新记忆将自动包含时态和置信度，旧记忆对应字段为空，不影响现有查询。

## 8. 兼容性说明

- **向后兼容**：v4.0 存储的记忆在 Neo4j 中 `tense` 和 `confidence` 为 `null`，查询时若未指定过滤条件，仍可正常召回。
- **Milvus 无影响**：向量检索仍基于 `content` 文本，语义不受影响。
